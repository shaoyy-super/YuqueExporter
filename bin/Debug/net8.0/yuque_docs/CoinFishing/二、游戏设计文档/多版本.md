1.多版本思路

所有多版本的功能都存在同一个基础分支内，版本分支基于基础分支。

不同版本需要的配置在各自版本分支配置。

不同版本需要开启的功能通过功能开启控制。

不同版本需要的活动通过活动开启功能控制。
