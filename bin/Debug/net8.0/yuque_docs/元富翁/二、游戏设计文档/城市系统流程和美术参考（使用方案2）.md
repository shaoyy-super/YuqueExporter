# 玩法流程
## 建筑系统
+ 玩家通过在棋盘上赚货币，在城市中升级建筑，单个建筑有等级，每次升级建筑逐渐完善。
+ 单个城市所有建筑升级完毕后解锁并进入下一个城市。

## 城市系统
+ 游戏中设置一张大地图，大地图由多个区域组成，每个区域内设置多个城市，此处的城市即对应建筑系统中的城市。
+ ![](https://cdn.nlark.com/yuque/0/2024/png/43554293/1718606201189-dc63425f-4ca7-4a72-aa41-f279b6907e74.png?x-oss-process=image%2Fformat%2Cwebp%2Fresize%2Cw_1125%2Climit_0)

# 包装设定
## 方案2
+ 没有明确的星球概念，直接运用时空概念，整个大地图上会有不同的大虫洞，每个大虫洞内会有多个小虫洞。
+ 每个小虫洞对应一个城市，因为时空概念，因此大虫洞内的小虫洞没有限制，可以为各式各样的内容，相比方案1会更加灵活。

### 参考
+ 大地图设置多个大型虫洞，虫洞样式类似时空裂缝，每个大虫洞上会有具象的主题图案。

![](https://cdn.nlark.com/yuque/0/2024/png/43554293/1719985602142-58ceefc0-4e28-4fe9-a716-17a574a408d7.png)

+ 进入大虫洞后，会有多个小虫洞，虫洞会显示当前城市的预览图。
+ 虫洞进行一定规律的排序，比如一条具体的路线，每个大节点会有多个小节点，类似下图的形式。
+ 后续结合剧情包装，每个小节点可以为单个小故事，也可以是每个大节点为一个完整的故事，内部有多个小节点的内容组合。

![](https://cdn.nlark.com/yuque/0/2024/png/43554293/1719985656602-0bea3114-3150-4fb4-b685-a14105d6723e.png)
